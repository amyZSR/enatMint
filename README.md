# eDMT Auto-Mint Tool

针对 [eDMT 协议](https://www.edmt.io)(eNAT)的本地浏览器自动 mint 工具。通过监控以太坊主网区块,自动构造并广播符合 eDMT 协议格式的 self-send 交易,以批量捕获历史区块的 burn 凭证。

> ⚠ **风险提示**:Hot Key 模式需要导入私钥到浏览器内存。私钥仅保存于当前页面会话期间,关闭即清除。**务必使用专门的 mint 钱包**,只放够 gas 的 ETH(0.1-0.5 ETH),不要使用主资产钱包。

## 1. 协议背景

### 1.1 eDMT 是什么

eDMT 是基于以太坊 EIP-1559 机制的 **burn 凭证协议**。每个 ETH 主网区块在被打包时会按规则销毁一定数量的 ETH(`baseFee × gasUsed`),这部分销毁的 ETH 永久从总供应中消失。

eDMT 协议把每个块的销毁量重新定义成一个可被认领的数字资产 — **eNAT**。

**核心规则:**

1. 协议起始块 `12,965,000`(EIP-1559 激活块,2021-08-05)
2. 每个块对应一张 eNAT,其内含 burn 量 = `floor(baseFee × gasUsed / 1e9)` gwei
3. Mint 方式:发送一笔 self-send 交易(`to == from`,`value == 0`),在 calldata 中携带特定 JSON
4. 所有权由 indexer 重放历史决定 — 同一个块第一个有效 mint 的发送者成为该块所有者
5. 链上没有合约,没有强制执行 — 完全由链下 indexer(edmt.io 跑的)按规则解析

### 1.2 Mint 交易格式

一笔合规的 mint 交易长这样:

```
To:    与 from 相同地址 (self-send)
Value: 0 ETH
Data:  data:,{"p":"edmt","op":"emt-mint","tick":"enat","blk":"<块号>"}
```

calldata 是 ASCII 文本编码后的 hex 字符串。链本身完全不知道 eDMT 协议存在 — 这只是一笔成功的 self-send,正常付 gas。但 indexer 看到这个特定 prefix 后,会按时间顺序判定 `blk` 字段所标识的块归属。

### 1.3 关键数据

| 指标 | 数值 |
|---|---|
| 全网历史 burn | 约 460 万 ETH(2026-02 Glassnode) |
| 已 mint 覆盖率 | 约 0.5%(1200 万块中已 mint 6 万张) |
| 最高 burn 时期 | Otherside Land mint(2022-04-30)单日 57,000 ETH |
| indexer 终结时延 | 12-15 分钟(主网 finality 后才计入持仓) |

## 2. 工具组成

### 2.1 界面分区

整个工具是一个**单文件 HTML**,所有功能在一个页面内。从上到下分 4 个 Plate:

| Plate | 名称 | 用途 |
|---|---|---|
| **0** | 警告区 | Hot Key 模式风险提示 |
| **I** | 连接区 | 钱包连接(MetaMask 或 Hot Key) |
| **II** | 实时状态区 | 当前块号、余额、累计花费 |
| **III** | 设置 + 日志区 | Mint 参数 + 实时活动日志 |
| **IV** | 持仓区 | edmt.io 链上确认的 wholes / 本次 session 战绩 |

### 2.2 两种连接模式

#### MetaMask 模式(默认 · 安全)

通过浏览器插件 MetaMask 连接钱包,每笔交易弹出确认窗口。

- ✅ 优点:私钥永远在 MetaMask 中,工具碰不到
- ❌ 缺点:每笔需要点确认,无法真正自动化
- 📌 适用:测试参数、谨慎打几笔

#### Hot Key 模式(高级 · 风险)

直接在工具中粘贴私钥,签名在内存中完成,使用 WebSocket RPC 直连节点。

- ✅ 优点:延迟 100-200ms,真正自动化批量 mint
- ❌ 缺点:私钥被工具读取(虽仅在内存中)
- 📌 适用:批量扫描 / 抢 mint

> ⚠ **Hot Key 模式必须使用专门的 mint 钱包**,该钱包只放够 gas 的 ETH(0.1-0.5 ETH 即可),不要放任何其他资产。即使私钥仅在内存,任何工具运行时的环境(浏览器扩展、剪贴板软件、屏幕录制)都可能形成数据风险。

## 3. 快速上手

### 3.1 准备工作

1. 一个干净的 Hot Key 钱包 + 0.1-0.5 ETH 作为 gas 预算
2. **Alchemy 免费账号**(申请 3 分钟)— 强烈推荐,稳定性远超公共 RPC
3. Chrome / Edge / Firefox 浏览器,新版本
4. 工具文件:`edmt-auto-minter.html`(双击在浏览器打开)

### 3.2 申请 Alchemy WebSocket

1. 打开 https://www.alchemy.com,Sign up(邮箱即可)
2. Dashboard → Create new app → Chain: **Ethereum**, Network: **Mainnet**
3. 点 **API Key** 按钮 → 复制 **WebSockets** URL
4. URL 格式:`wss://eth-mainnet.g.alchemy.com/v2/YOUR_KEY`

### 3.3 第一次连接

1. 打开工具,切到 **Hot Key Tab**
2. RPC URL 输入框粘贴 Alchemy WSS
3. Private Key 输入框粘贴 mint 钱包私钥(不带 `0x`)
4. 点 **Connect**。Plate II 显示 block# 一直跳动 = 成功
5. Plate II 应该显示余额 = 你充入的 ETH

### 3.4 测试一笔 mint

1. Plate III 选 **🐢 试水** 预设(成本 ~$0.10)
2. **Max attempts 改成 1**(只打 1 笔测试)
3. 打开 **Auto-mint** 开关 → 等几秒 → log 出现 `INCLUDED`
4. Plate IV 等 12-15 分钟点 **↻ Refresh**,确认 wholes +1

## 4. 核心参数详解

### 4.1 Block target offset(块偏移)

控制 mint 哪个块。规则:**目标块 = 当前块号 + offset**。

| Offset | 含义 |
|---|---|
| `0` | 抢实时新块(直播模式,最激烈竞争) |
| `-1000` | 打 1000 块前的块(~3.3 小时前) |
| `-50000` | 打 ~7 天前的块(捡漏推荐) |
| `-10277000` | 打 Otherside mint 那段(高 burn) |

> ℹ **SWEEP 模式下此参数被忽略** — sweep 直接用 from/to 指定的精确块号范围。

### 4.2 Min burn(最小 burn 阈值)

单位:**gwei**。block 的 burn 低于此值时,工具自动 SKIP 不发 tx。

| 输入数字 | 等于 | 等于 ETH | 约 USD 价值 |
|---|---|---|---|
| `1` | 1 gwei | 0.000000001 | 基本不过滤 |
| `100000000` | 1 亿 gwei | 0.1 ETH | $233 / 块 |
| `500000000` | 5 亿 gwei | 0.5 ETH | $1,165 / 块 |
| `1000000000` | 10 亿 gwei | 1 ETH | $2,330 / 块 |
| `2000000000` | 20 亿 gwei | 2 ETH | $4,660 / 块 |
| `5000000000` | 50 亿 gwei | 5 ETH | $11,650 / 块 |
| `10000000000` | 100 亿 gwei | 10 ETH | $23,300 / 块 |
| `50000000000` | 500 亿 gwei | 50 ETH | $116,500 / 块 |
| `100000000000` | 1000 亿 gwei | 100 ETH | $233,000 / 块 |

> ✓ **工具的 Min burn 输入框下方会实时显示解析结果**,确认数字位数无误。

### 4.3 Max fee per gas(最大 gas 价格)

单位:**gwei**。你愿意为每个 gas 单位支付的最大价格(包括 base fee + priority)。

**规则**:`maxFee` 必须 ≥ `baseFee + priorityFee`,否则 tx 被节点拒绝。

当前网络 base fee 约 0.15-0.2 gwei,推荐:

- 历史扫描:`30`(留 150x base fee buffer)
- 直播抢:`50-100`(防 base fee 飙升)

### 4.4 Priority fee(小费)

单位:**gwei**。直接给验证人的小费,决定 tx 在区块中的位置。

| Priority | 适用场景 |
|---|---|
| `0.1-0.5` | 历史扫描足够(没人和你抢) |
| `2-5` | 一般竞争(被人零星扫的区段) |
| `8-15` | 激烈竞争(Otherside 核爆区或直播抢) |
| `20+` | 极端抢前排(基本不需要) |

> ⚠ **priority fee 不要设 0**。某些验证人节点会过滤掉 priority < 0.001 gwei 的 tx,设 0.1 是安全平衡点。

### 4.5 Gas limit

单笔 mint tx 的 gas 上限。eDMT mint 因为 calldata 不长,实际消耗 22,000-25,000 gas。

**推荐设 `35000`**(留 buffer 防 `intrinsic gas too low` 错误)。

### 4.6 Rate limit(每分钟最多发几笔)

60 秒滑动窗口内的 tx 数量上限。直播模式默认 5/60s 防 nonce 撞车。

> ℹ **SWEEP 模式自动绕过 rate limit**,改用全局 1 笔/秒硬限速,无需调整此参数。

### 4.7 Max attempts(最多尝试笔数)

达到此值工具自动停止。建议根据预算反推:

- 试水:`20-50`
- 中等:`100-300`
- 量产:`500-1000`

### 4.8 Max ETH spend(最大 ETH 消费)

达到此值工具自动停止。是 attempts 之外的双重保险。

- 测试:`0.005` ETH (~$12)
- 中等:`0.05` ETH (~$117)
- 量产:`0.2` ETH (~$466)

## 6. SWEEP 模式(推荐)

### 6.1 SWEEP 是什么

Rolling offset 模式每出新块才打一笔(12 秒 1 笔,慢)。**SWEEP 模式则是直接给定一个块号区段(from-to),工具按 1 笔/秒持续遍历**,几乎不需要等。

适用于扫描历史「金矿区段」(高 burn 历史时期),效率比 rolling 高 12 倍。

### 6.2 启用 SWEEP

1. Plate III 找到 **SWEEP 模式 · 横扫指定块号区段** 复选框
2. 勾上后,下方 4 个新字段(from / to / 方向 / 并发)激活
3. **Sweep 起始块号 / Sweep 结束块号** — 填具体块号(不是 offset)
4. **方向**:推荐「⇄ 随机」(避开跟其他人撞车)
5. **并发**:默认 1。全局硬限速 1 笔/秒,改大也无意义
6. Min burn 等普通参数仍按 Plate III 顶部设置生效
7. 打开 **Auto-mint** 开关 → 自动开始扫描
8. 扫完整段自动停止 + log `SWEEP DONE`

### 6.3 进度面板

启动 SWEEP 后,Plate III 会出现实时进度行:

```
SWEEP · 进度 142/29500 (0.5%) · 剩 29358 · 在飞 1 · 已发 142
```

- **进度**:已处理 / 总块数
- **剩**:队列中等待处理的
- **在飞**:当前正在 fetch 或 mint 的
- **已发**:已经发出的 mint tx 数(含 SKIP)

### 6.4 日志类型对照

| 日志类型 | 颜色 | 含义 | 处理 |
|---|---|---|---|
| `INCLUDED #X` | 🟢 | tx 上链成功 | 等 indexer 确认归属 |
| `PENDING` | 🟡 | tx 已广播,等上链 | 等 1-2 个块 |
| `SWEEP SKIP burn<min` | 🔴 | block burn 低于阈值,跳过 | 正常,不耗 gas |
| `SWEEP SKIP · 块无数据` | 🔴 | RPC 拉不到该块 | RPC 抖动,自动重试 |
| `RATE-LIMIT` | 🟡 | 60s 内已达发送上限 | 改大 rate limit 或用 SWEEP |
| `REVERTED` | 🔴 | tx 上链但失败 | 极少见,gas limit 过低 |
| `DROPPED` | 🔴 | tx 进 mempool 后被踢 | max fee 太低被超过 |

## 8. 实战策略

### 8.1 单价 vs 总量

eDMT 排行榜按「**持有 burn 总量**」排,不是按张数。两种打法:

**数量打法**:
低阈值(0.1-0.3 ETH)+ 长尾区段。每张 burn 小,但量大。适合冲张数排名。

**burn 打法**:
高阈值(2-10 ETH)+ 高 burn 区段。每张 burn 大,数量少。适合冲 burn 总量排名。

> ✓ **推荐 burn 打法**。排行榜前 5 名平均每张 burn 都在 100+ ETH,他们打的是 Otherside 核爆区。账面价值是 burn × ETH 价格,而不是张数 × 价格。

### 8.2 避开竞争的方法

1. 用「**随机**」方向 — 别人都从 from 顺序扫,你打乱顺序就避开正面
2. 一段最多打 200 笔就切下一段 — 防对手发现你后切到同一段
3. **rotate offset** — 比如每 200 笔切一次:`-50000` → `-100000` → `-300000` → `-50000`…
4. 挖**冷门时期** — Inscriptions、FTX 崩盘、BAYC origin 等关注度低于 Otherside
5. 运行时段错峰 — 美国白天竞争最激烈,亚洲深夜对手少

### 8.3 阈值校准

跑 30-50 笔后看实际命中的 burn 分布:

1. 命中的 burn 都集中在阈值附近 → 阈值适中 ✓
2. 命中的 burn 都远高于阈值 → 阈值太低,提一档过滤精品
3. 几乎不命中 → 阈值太高 / 这段已被扫,降一档或换段
4. INCLUDED 比例(指 mined / attempted)< 30% → 这段在被竞争,priority 加 2-5 gwei 或换段

### 8.4 多账号并行

浏览器开多个无痕窗口,每个窗口独立连一个 Hot Key 钱包,跑不同 offset 区段。Alchemy 免费版单 API key 每秒 25 请求,跑 5-10 个账号无压力。

> ⚠ **多账号必须打不同区段或不同 offset**,否则两个账号会自己跟自己抢同一个块,白浪费 gas。

## 9. 故障诊断

### 9.1 常见错误对照

| 错误关键字 | 原因 | 解决方案 |
|---|---|---|
| `nonce too low` | nonce 撞车 | 降并发到 1 / 重连钱包 |
| `replacement underpriced` | 同一 nonce 重复发 | 等 1-2 个块自动恢复 |
| `429 Too Many Requests` | RPC 限流 | 换 Alchemy / 降速 |
| `insufficient funds` | ETH 余额不够付 gas | 充 ETH |
| `intrinsic gas too low` | gas limit 设太低 | 改成 35000+ |
| `WebSocket closed` | RPC 掉线 | 刷新页面重连 |
| `No block data` | RPC 无 archive 历史 | 换 Alchemy / Infura |
| `CORS proxy 失败` | 免费 proxy 挂了 | 等几分钟自动恢复 / 手动填 |
| `RATE-LIMIT` | 工具自身限速 | 用 SWEEP 模式自动绕过 |

### 9.2 工具卡死时

1. 先点 **STOP** 关闭 Auto-mint
2. F12 打开浏览器控制台,看红色错误信息
3. `Ctrl+Shift+R` 强制刷新页面(清浏览器缓存)
4. 重连钱包(Hot Key 模式需重新粘私钥)
5. 从 SWEEP 队列继续:已 mint 的块不会重打,从下个未处理块开始

### 9.3 持仓不同步

工具内的「本次 session」是当前页面 JS 内存中的累计,**关页面就清空**。Plate IV「持有 wholes」是从 edmt.io 拉的链上确认数据,需手动点 ↻ Refresh。

常见情况:刚 mint 的 wholes 不立刻显示在持仓 — 因为需要 12-15 分钟 finality。如果 30 分钟后仍不更新,可能是 CORS proxy 全挂,会自动切换到手动填入框。

## 10. ROI 计算与变现

### 10.1 协议层价值 vs 真实价值

工具显示的「持有 burn」是**协议含义**,不等于市场价。

```
持有 burn 250 ETH × $2,330 = $582,500
```

这个数字是「**理论账面**」— 假设市场按 1:1 burn 价格收购的话。实际能卖多少取决于:

1. wENAT / bENAT 二级市场买单深度
2. 是否有人愿意按折扣收(常见 0.05x-0.5x burn 价格)
3. 具体块号是否稀有(纪念块、整数块、特殊事件块溢价)

### 10.2 真实成本计算

```
单张获取成本 = 累计 spent USD / 拥有 wholes 数量
```

举例:

```
累计 spent: $50
拥有 wholes: 331 张
单张成本: $50 / 331 = $0.151 / 张
```

### 10.3 边际成本(更准)

## 11. 安全与风险

### 11.1 私钥风险

- Hot Key 模式下私钥保存在浏览器 JS 内存(不写入硬盘 / localStorage)
- 关闭页面或浏览器后立刻清除
- 但运行期间任何能读取浏览器内存的恶意软件理论上可获取
- **强烈建议**:专门的 mint 钱包,只放够 gas 的 ETH(0.1-0.5 ETH)
- **绝不**使用主资产钱包 / 含大量代币 / 持有 NFT 的钱包

### 11.2 协议风险

- eDMT 是社区协议,无 DAO 无治理。indexer 由 edmt.io 单方运营
- 理论上 indexer 可单方面修改规则、回滚所有权、关闭服务
- 没有合约保护 — 你「持有」的 wholes 不在链上,只在 edmt.io 数据库
- 二级市场流动性极不稳定 — wENAT 可能某天没人接盘

### 11.3 Gas 风险

- base fee 突然飙升时,如果 max fee 设得低,你的 mint tx 全部被踢
- Hot Key 模式下意外参数错误 + max attempts 设高 = 短时间烧光预算
- **永远先设较小的 max attempts 和 max ETH spend 测试**

### 11.4 投入建议

按「测试 → 验证 → 量产」三阶段:

| 阶段 | 投入预算 | 目标 |
|---|---|---|
| **测试** | $5-10 | 验证工具运行正常 / 单张成本 |
| **验证** | $20-50 | 验证特定区段 ROI / 二级市场流动性 |
| **量产** | $100-500 | 确认 ROI 后大规模 sweep |

> ✓ **完整流程**:测试 1-3 笔 → 等 15 分钟 indexer 确认 → 测试一段 SWEEP → 拉持仓数据 → 看二级市场 → 决定是否量产。
